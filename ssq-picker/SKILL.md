# 双色球选号 Skill

> **本文档结构**：🔧 操作流程（必读）→ 📖 参考资料（按需查阅）

## 数据文件

- **缓存文件**：`{SKILL_DIR}/data/draws.jsonl`（JSON Lines 格式，每行一条开奖记录）
- 首次使用自动创建 `data/` 目录并全量拉取；后续自动增量追加
- 文件格式（每行一条 JSON 对象）：
  ```json
  {"issue": "2026060", "date": "2026-05-28", "reds": [3, 4, 14, 22, 23, 33], "blue": 4}
  ```
  - `issue`：7 位期号字符串（如 "2026060"）
  - `date`：日期字符串 "YYYY-MM-DD"
  - `reds`：6 个整数数组 [1-33]，已排序
  - `blue`：单个整数 [1-16]

---

## 执行流程总览

```
用户说"选号/买彩票/推荐号码"？
├── Step 0：运行 update_data.py 更新数据（失败则 fallback）
├── Step 1：随机选择 模式一 or 模式二
│   ├── 模式一：analyze.py 出候选 → 玄学从候选中定夺 6+1
│   └── 模式二：玄学感应候选池 → scientific_pick.py 从候选中精选 6+1
├── Step 2：确定目标期数（最新期+1 / 跨年处理 / 用户指定）
└── Step 3：按标准输出格式展示结果
```

用户说"更新数据/开奖/对奖"？→ 直接运行 update_data.py 并报告结果。

---

## 基础规则

- 红球：从 01~33 中选 6 个，不重复，开奖时顺序无关
- 蓝球：从 01~16 中选 1 个
- 一注 = 6 个红球 + 1 个蓝球

## 🔧 操作流程

### Step -1：数据初始化（首次使用）

检查 `{SKILL_DIR}/data/draws.jsonl` 是否存在：

```bash
ls "{SKILL_DIR}/data/draws.jsonl" 2>/dev/null && echo "EXISTS" || echo "MISSING"
```

若 **MISSING** → 创建目录并全量拉取：

```bash
mkdir -p "{SKILL_DIR}/data"
python3 {SKILL_DIR}/scripts/update_data.py --init
```

`--init` 模式会拉取全部历史数据写入 `draws.jsonl`。完成后进入正常流程。

### Step 0：数据新鲜度检查

```bash
python3 {SKILL_DIR}/scripts/update_data.py "{SKILL_DIR}/data/draws.jsonl"
```

脚本自动判断是否需要增量更新：已是最新则秒级返回 `"已是最新"`；有新数据则追加到 JSONL 末尾；网络不通则跳过用本地缓存。

### 错误处理（fallback 规则）

当任何脚本返回 `{"error": ...}` 时：

1. **数据更新失败** → 告知用户"数据可能非最新"，继续用本地数据分析
2. **分析脚本失败** → 回退到**纯随机选号**（01-33 随机 6 红 + 01-16 随机 1 蓝），标注 `⚠️ 本次为随机模式（分析服务暂不可用）`
3. **不要重试** → 一次失败即 fallback，保证用户总能拿到号码

### Step 1：随机选模式

| 模式 | 第一阶段（出候选池） | 第二阶段（精选 6+1） |
|------|------|------|
| **模式一：科学→玄学** | 🔬 analyze.py 科学分析 | 🔮 玄学从候选中定夺 |
| **模式二：玄学→科学** | 🔮 玄学感应候选池 | 🔬 scientific_pick.py 科学精选 |

两种模式核心逻辑相同，只是角色互换。每次**随机选一种**，不要总是同一种。

---

#### 模式一：科学→玄学

**第一阶段 —— 运行科学分析脚本：**

```bash
python3 {SKILL_DIR}/scripts/analyze.py "{SKILL_DIR}/data/draws.jsonl"
```

脚本每次随机选择分析策略（期数、筛选条件、3-6 种维度组合），输出 JSON 候选号码。
可用维度及原理详见 `{SKILL_DIR}/references/scientific-systems.md`。

输出示例：
```json
{
  "strategy": "分析最近100期数据，筛选条件：全部期数，实际分析100期，使用维度：冷热号统计、遗漏值分析、区间分布分析",
  "candidate_reds": [3, 5, 8, 12, 14, 17, 20, 22, 25, 27, 29, 31, 32, 33, 16],
  "candidate_blues": [2, 5, 9, 12, 14],
  ...
}
```

**第二阶段 —— 玄学从候选中选出最终 6+1：**

从「玄学体系列表」中**随机选一种**（详见 `{SKILL_DIR}/references/divination-systems.md`），基于其逻辑从候选池定夺最终号码。

规则：
- 红球必须从候选红球中选取 6 个，从小到大排列
- 蓝球必须从候选蓝球中选取 1 个
- 必须根据当前日期独立推演，严禁套用示例中的具体数字
- 解读简短有趣（2-3句）

---

#### 模式二：玄学→科学

**第一阶段 —— AI 玄学感应候选池：**

从「玄学体系列表」（见 `{SKILL_DIR}/references/divination-systems.md`）中**随机选一种**，由 AI 运用该体系的逻辑推演：

- 候选红球：**12~20 个**（从 1-33 中选）
- 候选蓝球：**4~10 个**（从 1-16 中选）
- 推演过程简短有趣（2-3句），自洽即可
- **必须根据当前日期独立推演，严禁套用示例中的具体数字**

**第二阶段 —— 运行科学精选脚本：**

```bash
python3 {SKILL_DIR}/scripts/scientific_pick.py "{SKILL_DIR}/data/draws.jsonl" \
  --reds "候选红球逗号分隔" --blues "候选蓝球逗号分隔"
```

脚本对候选池多维评分精选最优 6+1。
可用维度详见 `{SKILL_DIR}/references/scientific-systems.md`。

---

### Step 2：确定预测期数

| 场景 | 规则 |
|------|------|
| 默认 | 最新一期期号 +1（如最新 `2026040` → 预测 `2026041`） |
| 跨年 | 当前日期已入新一年但最新期号属上一年 → `新年份001` |
| 用户指定 | 使用用户指定的期数 |

读取最新期号方法：JSONL 最后一行即为最新记录。

### 独立功能：仅更新数据
